# Στόχος συστήματος — Πλακέτα ελέγχου καυστήρα Vevor

Έγγραφο αναφοράς για νέα κατασκευή (Rev 2). Ενημερώθηκε με φωτογραφίες πίνακα Vevor (2026-05-18).

**Μεγάλη εικόνα (καζάνι, 80m² UFH, ηλιακός, modulation):** δείτε `ΟΡΑΜΑ_ΣΥΣΤΗΜΑΤΟΣ.md`.

---

## 1. Τι ελέγχουμε (και τι ΟΧΙ)

| Ελέγχουμε | Δεν αντικαθιστούμε |
|----------|-------------------|
| Πίνακα **ON/OFF** του Vevor (με ρελέ) | Την κύρια πλακέτα καυστήρα |
| «Ρύθμιση θερμοκρασίας» (με ψηφιακό pot πάνω στον μοχλό/κύκλωμα pot) | Το firmware του LCD του Vevor |
| Κυκλοφορητές νερού (ρελέ) | — |
| Θερμοκρασίες νερού (DS18B20) | — |
| Δική μας οθόνη 20×4 (παρακολούθηση) | — |

Ο καυστήρας μένει **κινέζικος 5kW 12V diesel heater** (βλ. `HARDWARE_ΚΑΥΣΤΗΡΑΣ.md`) με τη **δική του συμπεριφορά**· εμείς τον «οδηγούμε» από έξω.

## 2. Πίνακας Vevor (από φωτογραφίες)

**Πλακέτα:** πράσινο PCB, κεντρική οθόνη, σήμανση **LCDMAX V2 / 270502**.

### Κουμπιά

| Θέση | Ρόλος (επιβεβαιωμένο από εσένα) |
|------|----------------------------------|
| **Πάνω σειρά — 3 κουμπιά — ΜΕΣΑΙΟ** | **ON / OFF** (power) |
| Πάνω — αριστερά / δεξιά | Άλλες λειτουργίες (±, menu — όχι για την τρέχουσα λογική) |
| Κάτω σειρά — 3 κουμπιά | Πρόσθετα (κάτω μέσα: header 6 pin) |

### Η επέμβασή σου (φωτό `photos/vevor_panel_wires_on_off.png`)

- **Δύο λεπτά καλώδια** κολλημένα στα **δύο πόδια** του **μεσαίου πάνω** μικροδιακόπτη (NO/NC ζεύγος — το ρελέ «πατάει» ανάμεσα στα ίδια σημεία που κλείνει το κουμπί με το δάχτυλο).
- Τα καλώδια πήγαιναν σε **ρελέ 5V module** (στη δική σου πλακέτα H.T.C.).
- Το ρελέ **κλείνει επαφή ~2,5–3 s** → ισοδύναμο **παρατεταμένο πάτημα** ON/OFF.

**Στον κώδικα:** `RELAY_VEVOR_BTN`, `delay(2500)` — ίδια ιδέα με τα 3 s που θυμάσαι.

### Οθόνη Vevor (φωτό `photos/vevor_panel_lcd_working.png`)

- Δείχνει π.χ. **35°C**, **P 13V**, εικονίδια — αυτό είναι **UI του καυστήρα**, όχι το δικό μας LCD 20×4.
- Η **δική μας** οθόνη δείχνει Boiler / Prosagogi / Epistrofi / Katastasi.

## 3. Συμπεριφορά καυστήρα (γιατί χρειάζεται ρελέ ON/OFF)

```text
Θέρμανση ON  →  φλόγα + ανεμιστήρας
Φτάνει setpoint  →  σβήνει φλόγα, ανεμιστήρας συνεχίζει (δροσερό ~2–3 min)
Εσωτερική θερμ. ~30°C  →  σβήνει και ανεμιστήρας
Για ξανά-έναρξη  →  χειροκίνητο ON/OFF (ή δικό μας πάτημα 2,5–3 s)
```

**Πρόβλημα:** Αν πατήσεις ON **ενώ** ο ανεμιστήρας ακόμα «σβήνει καυτό», συχνά **δεν πιάνει**.

**Λύση (που δούλεψε):** Ο **δικός σου κύκλος** (35°C / 42°C hysteresis) αποφασίζει πότε «πατάμε» ON/OFF — όχι εσύ χειροκίνητα.

## 4. Δομή συστήματος H.T.C. (τι φτιάχνουμε ξανά)

```text
                    ┌─────────────────────────────────┐
  12V τροφοδοσία ──►│  H.T.C. πλακέτα (carrier)       │
                    │  • Step-down 5V (λογική)        │
                    │  • Step-down 5V (ρελέ/modules)  │
                    │  • MCU (Nano παλιά / ESP32 νέα) │
                    │  • LCD 20×4 I2C (δική μας)      │
                    │  • Ψηφιακό pot → μοχλός Vevor   │
                    │  • Ρελέ → ON/OFF Vevor          │
                    │  • Ρελέ(ά) → κυκλοφορητές       │
                    │  • 1-Wire → θερμομετρήσεις      │
                    └──────────┬──────────────────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         ▼                     ▼                     ▼
   DS18B20 ×3            Πίνακας Vevor          Κυκλοφορητές
   (σωλήνες)             (2 καλώδια ON/OFF)      (ρελέ 220V)
```

## 5. Λογική firmware (επιβεβαιωμένη — βάση Rev 1)

### 5.1 Είσοδοι

- **3× DS18B20** σε ένα bus: Boiler (0), Prosagogi (1), Epistrofi (2).

### 5.2 Έξοδοι (minimum που δούλεψε)

| Έξοδος | Συνθήκη / ενέργεια |
|--------|-------------------|
| **Ψηφιακό pot** | FULL / MID / IDLE / OFF ανά θερμοκρασία Boiler + κατάσταση κύκλου |
| **Ρελέ ON/OFF** | Παλμός **2,5–3 s** μόνο όταν **αλλάζει** heating ON↔OFF |
| **Ρελέ αντλίας** | ON αν Boiler > 25°C |

### 5.3 Hysteresis κύκλου

- Boiler **< 35°C** → heating cycle ON  
- Boiler **≥ 42°C** → heating cycle OFF  

### 5.4 Pot τιμές (από δουλεύοντα sketch)

| Όνομα | SPI value | Σχόλιο |
|-------|-----------|--------|
| FULL  | 128       | ισχυρή θέρμανση |
| MID   | 197       | |
| IDLE  | 230       | χαμηλή πριν το OFF |
| OFF   | 250       | όταν όχι heating cycle |

## 6. Rev 1 (παλιά πλακέτα) vs Rev 2 (στόχος)

| Θέμα | Rev 1 (έκανες) | Rev 2 (πρόταση) |
|------|----------------|-----------------|
| MCU | Arduino Nano | **ESP32 30 pin** |
| PCB | H.T.C. 2026, Nano footprint | Νέο σχέδιο ή προσαρμογή |
| Fan timing | Σχεδιασμένο + **πρόχειρη καλωδίωση** (φωτό fan) — όχι στον αποθηκευμένο κώδικα | **Να μπει** στο Rev 2 (GPIO / tach / χρονόμετρο) |
| 2η αντλία | 2ch relay στο PCB | Να οριστεί |
| Buzzer | Footprint | Να οριστεί |
| Πίεση νερού | Footprint | Προαιρετικό |

---

## 7. Κανόνες καλωδίωσης (ασφάλεια / χωρίς λάθη)

1. **ON/OFF:** Ρελέ **παράλληλα** στο ζεύγος επαφών του μεσαίου κουμπιού — **όχι** σειριακά με την οθόνη. Μικρό ρεύμα, αλλά σταθερή επαφή.
2. **Pot:** Δύο καλώδια στο **ίδιο σημείο** που συνδέεται ο μοχλός θερμοκρασίας (όπως πριν) — **μην** μπερδεύεται με ON/OFF.
3. **Ρελέ 220V (αντλίες):** Ξεχωριστό 5V, καλή μόνωση, **όχι** κοινό GND με σημάτα χωρίς σχέδιο.
4. **Χρόνος πάτημας:** **2,5–3 s** — μικρότερο μπορεί να μην «πιάσει», μεγαλύτερο διπλό toggle.
5. **Δοκιμή:** Πάντα χωρίς καύσιμο πρώτα (ρελέ μόνο, Serial + LCD).

### Πρόχειρο κύκλωμα (`photos/prototype_breadboard_relay_active.png`)

Στιγμή λήψης: **το ρελέ μόλις έχει ανάψει** (κόκκινο LED module ON) → `toggleVevorPower()` / αλλαγή heating cycle.

| Τι φαίνεται | Ρόλος |
|------------|--------|
| **Arduino Nano** + USB | MCU — ίδιο με τον κώδικα |
| Πράσινο perfboard + **κλέμματα** (μπλε/πράσινα) | Πρόχειρη καλωδίωση, όχι H.T.C. PCB |
| **1-channel relay module** (μπλε, LED **ανοιχτό**) | = `RELAY_VEVOR_BTN` (D7) — πατάει ON/OFF Vevor |
| **Buck converter** ένδειξη **12.0** | Τροφοδοσία ~12V (όπως step-down στην πλακέτα) |
| **LCD I2C** (μεγάλη, μπλε) — γραμμές «γεμάτες» | Δική μας οθόνη 20×4· εκεί ίσως boot / χωρίς κείμενο ακόμα |
| **Πίνακας Vevor** συνδεδεμένος (05°C, P 13V) | Το panel που πατάει το ρελέ |
| **Μαύρα καλώδια** Vevor ↔ ρελέ | Ίδια επέμβαση με φωτό ON/OFF |

**Δεν φαίνεται καθαρά σε αυτή τη φωτό (ίσως εκτός κάδρου):** ψηφιακό pot (SPI), DS18B20, 2ο ρελέ αντλίας, buzzer — στον κώδικα υπάρχουν, στην πλακέτα H.T.C. υπήρχαν θέσεις.

### Επέμβαση ανεμιστήρα (`photos/vevor_fan_splice_yellow_module.png`)

**Τι βλέπουμε (όπως το περιγράφεις):**

```text
[Κίτρινο module Vevor στο σώμα καυστήρα]
    │
    ├─ Δεξιά κόκκινη κλέμα (2-pin): ΚΟΚΚΙΝΟ + ΜΑΥΡΟ  ──►  μεγάλο στρογγυλό fan (αριστερά)
    │         │
    │         └── εδώ έκανες ΚΟΨΙΜΟ / παράλληλη σύνδεση
    │                   │
    │                   └── τα δικά σου R+B ──► Fan Controller στην H.T.C.
    │                        (H6 / U12 στο PCB)
    │
    └─ Αριστερή κλέμα: 2× καφέ καλώδια (άλλο κύκλωμα — π.χ. αισθητήρας/άλλη έξοδος)
```

**Γιατί το έκανες:** Ο Vevor μετά το setpoint σβήνει τη **φλόγα** αλλά ο **ανεμιστήρας** μένει ON μέχρι να πέσει η εσωτερική θερμοκρασία (~2–3 min). Αν πατήσεις ON/OFF **ενώ** ακόμα «σβήνει καυτό», συχνά **δεν πιάνει**. Θέλεις να **ξέρεις** πότε ο fan τρέχει (ή να τον ελέγχεις) από τη **δική σου** πλακέτα.

**Στο PCB H.T.C.:** θέσεις **FAN CONTROLLER**, **H6-CONTROL SERVO MOTOR**, **U12 FAN CONTROLLER** — αυτό ταιριάζει με αυτή την καλωδίωση (όχι με τον παλιό `.ino` που δεν έχει fan code).

**Για Rev 2 (ESP32) — να οριστεί πριν το σχέδιο:**

| Ερώτηση | Επιλογές |
|---------|----------|
| Μόνο **παρακολούθηση** (fan ON/OFF ή RPM); | Digital input / tach |
| Και **έλεγχος** τροφοδοσίας fan; | Ίδιο module + relay ή MOSFET (προσοχή ρεύματος) |
| Κοινό GND με 12V; | Υποχρεωτικό σαφές σχέδιο |

**Προσοχή καλωδίωσης:** Κόψιμο στο ζεύγος R/B προς fan = **παράλληλα** με το εργοστασιακό κύκλωμα· το fan controller της μητρικής πρέπει να **μην** περνάει υπερβολικό ρεύμα από GPIO — συνήθως **sense** (τάση/παρουσία 12V) ή **opto**, όχι απευθείας drive motor από pin ESP32.

Όταν βρεις άλλες φωτό (pot, αντλίες, πίσω πλευρά perfboard), βάλ’ τες στο `photos/`.
