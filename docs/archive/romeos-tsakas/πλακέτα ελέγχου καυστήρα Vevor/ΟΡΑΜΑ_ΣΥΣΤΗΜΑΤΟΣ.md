# Όραμα συστήματος — Λέβητας πετρελαίου + H.T.C. (Rev 2)

Έγγραφο σκεπτικού (όχι τελικό σχέδιο). Θα μελετηθεί **ένα-ένα** κατά την κατασκευή της νέας πλακέτας.

---

## 1. Τελικός στόχος (όχι «κινέζικο kit ως έχει»)

```text
[Ήλιος 18 κενού] ──► [Boiler 160L τριπλής ενέργειας]
                              ▲
                              │  (απλό καζάνι / λέβητας — όχι κλειστό δοχείο πίεσης)
                              │
                    [Καυστήρας 5kW — μόνο πυρήνας + fan]
                    (χωρίς πλαστικά περίβλημα kit)
                              │
                    [H.T.C. / ESP32 — αυτόματος έλεγχος]
                              │
         [Αισθητήρες T] ── δωμάτιο, επιστροφή, προσαγωγή, ηλιακός, κ.λπ.
                              │
                    [Ενδοδαπέδια 80 m² — πολύ καλή μόνωση]
```

| Στοιχείο | Περιγραφή |
|----------|-----------|
| **Κτίριο** | **Σπίτι φίλου**, ~**80 m²**, **ενδοδαπέδια**, **πολύ καλή μόνωση** (δάπεδο, τοίχοι, οροφή)· υδραυλικό από Romeo |
| **Οθόνη χώρου** | `romeos-display-v4` (θερμοστάτη Viewe)· δεύτερη οθόνη στο κουτί για τον φίλο — ξεχωριστό firmware, ίδια εγκατάσταση |
| **Ηλιακός** | Κενού, **18 λάμπες** (αρχικά· το σύστημα σχεδιάζεται γύρω από 18) |
| **Αποθήκευση** | **Boiler τριπλής ενέργειας 160 L** |
| **Καζάνι / λέβητας** | **Δική σου κατασκευή** — απλή, **όχι** κλειστή πίεση· πάνω **συμπυκνώνεται** ο καυστήρας |
| **Καυστήρας** | Από AliExpress 5kW 12V· **αφαιρούνται πλαστικά** (μένει ουσιαστικά **fan + καύση**)· μηχανικές βελτιώσεις για αντοχή |

**Σκοπός ελέγχου:** Να **μην** χρειάζεται χειροκίνητα ON/OFF ή «πήγαινε δες τον καυστήρα» — όλα από **αισθητήρες** + λογική στη **νέα πλακέτα**.

## 2. Τι έκανες ήδη (επιβεβαιωμένο — Rev 1)

| Πρόβλημα καυστήρα | Επέμβαση | Στον παλιό `.ino` |
|------------------|----------|-------------------|
| Πρέπει να πατήσεις ON/OFF | Ρελέ παράλληλα στο κουμπί, 2,5–3 s | ✅ D7 |
| Θερμοκρασία μόνο από μοχλό | Ψηφιακό pot (SPI) | ✅ D10, FULL/MID/IDLE/OFF |
| Δεν ξέρεις αν τρέχει ακόμα fan | Κόψιμο R/B → fan controller PCB | ❌ όχι στον κώδικα |
| Παρακολούθηση | 3× DS18B20 + LCD 20×4 | ✅ |
| Κυκλοφορητής | 1 ρελέ από T boiler | ✅ D4 |

Αυτό **δούλεψε** στο πρόχειρο κύκλωμα — είναι η **βάση**, όχι ο τελικός στόχος.

## 3. Το βασικό μειονέκτημα που θέλεις να λύσεις (νέο θέμα)

### Συμπεριφορά «κινέζικου» panel

- Η **«ταχύτητα» / ισχύς** (επίπεδα **H** στο LCD, π.χ. H2, H6) είναι **σταθερή ρύθμιση**.
- Αν το βάλεις στο **H2**, ο καυστήρας δουλεύει **πάντα** σε αυτό το επίπεδο — **ανεξάρτητα** αν το νερό είναι κρύο ή ζεστό, αν ζητάει το δάπεδο περισσότερη ή λιγότερη θερμότητα.
- **Δεν κάνει modulation** (δεν ανεβοκατεβάζει μόνος του την «φλόγα» / αντλία καυσίμου / duty).

### Τι έχεις ψάξει

- Δύσκολη πρόσβαση στο εσωτερικό του panel για να αλλάξεις **Hz / PWM** της αντλίας καυσίμου από το firmware του panel.
- Άρα η λύση δεν είναι «πειράζω το menu H» — είναι **παράκαμψη** όπως με pot και ON/OFF.

### Στόχος Rev 2 (ακόμα ανοιχτό — θα συζητηθεί)

**Να οδηγείται η «ισχύς» από τους δικούς σου αισθητήρες**, όχι από σταθερό H:

| Πιθανή κατεύθυνση | Σημειώσεις |
|------------------|-----------|
| Βαθμιαίο **pot** (όπως τώρα) αλλά με **πιο λεπτά βήματα** και λογική από ΔT / T επιστροφής | Εύκολο επέκταση κώδικα |
| **PWM / relay** στην τροφοδοσία αντλίας καυσίμου | Χρειάζεται reverse engineering + ασφάλεια |
| **Εξωτερικός** controller καυσίμου + αντλία (μεγάλη αλλαγή) | Άλλο project |
| **Duty cycle** ON/OFF καυστήρα (χονδρικό modulation) | Απλούστερο· φθορά / απόδοση; |

→ **Θα το ανοίξουμε σε ξεχωριστό κεφάλαιο** πριν σχεδιάσουμε hardware.

## 4. Άξονες ελέγχου (για τη νέα πλακέτα)

| # | Άξονας | Στόχος | Κατάσταση |
|---|--------|--------|-----------|
| A | **ON/OFF καυστήρα** | Αυτόματο από κύκλο θέρμανσης + ασφάλειες | ✅ αποδειγμένο |
| B | **Setpoint / «ισχύς»** (pot) | Από θερμοκρασίες νερού / υστερότητα ηλιακού | ✅ αποδειγμένο, βελτίωση |
| C | **Fan phase** | Ξέρεις αν ακόμα «σβήνει καυτό» — μην πατήσεις ON πρόωρα | 🔶 hardware OK, firmware Rev 2 |
| D | **Modulation ισχύος (Hz/H)** | Όχι σταθερό H2 — ανά ζήτηση | 🔴 μελέτη / R&D |
| E | **Κυκλοφορητές** | UFH + boiler + ηλιακός (πολλά ρελέ / BMS) | 🔶 επέκταση |
| F | **Ηλιακός / 160L** | Προτεραιότητα καυσίμου vs ηλιακό | 🔶 λογική BMS |
| G | **Ασφάλειες** | Υπερθέρμανση, απώλεια ροής, buzzer, watchdog | 🔶 Rev 2 |

## 5. Προτεινόμενη σειρά μελέτης (ένα-ένα)

1. **Υδραυλικό / καζάνι** — πού μπαίνουν οι αισθητήρες (T1..Tn), ροή UFH. → **🔜 ΕΠΟΜΕΝΟ** (`ΕΠΟΜΕΝΗ_ΣΥΖΗΤΗΣΗ_ΥΔΡΑΥΛΙΚΟ.md`)  
2. **Καυστήρας μηχανικά** — τι μένει, πού μπαίνει, ψύξη fan.  
3. **Επαναληπτήματα Rev 1** — ON/OFF, pot, fan sense (σχηματικό).  
4. **Modulation (H / Hz)** — τι μπορούμε πραγματικά να παρακάμψουμε (με μέτρηση στο κίτρινο module / αντλία).  
5. **BMS λογική** — 80 m² UFH + 18 λάμπες + 160L (π.χ. T επιστροφής > X → ανάβει καυστήρα).  
6. **ESP32 Rev 2 PCB** — pins, 12V, ρελέ, LCD/Wi‑Fi, inputs.  
7. **Firmware** — χωρίς `delay(2500)` block· states· logging.  
8. **Δοκιμές** — bench → με καυστήρα → με νερό.

## 7. Ανοιχτά ερωτήματα (για επόμενες συζητήσεις)

- Ποιες **θερμοκρασίες** ορίζουν «ανάβω καυστήρα» vs «μόνο ηλιακός»;  
- Πώς συνδέεται **boiler 160L τριπλής ενέργειας** στον κύκλο (T sensor points);  
- Modulation: **αρκεί** βελτιωμένο pot + hysteresis ή χρειάζεται **πραγματικό Hz**;  
- Χρειάζεται **ασφάλεια πυρκαγιάς / CO** (ανεξάρτητο από H.T.C.);  
- ESP32: μόνο τοπικό LCD ή και **Wi‑Fi / remote** (υπήρχε footprint Wi‑Fi στο PCB).
