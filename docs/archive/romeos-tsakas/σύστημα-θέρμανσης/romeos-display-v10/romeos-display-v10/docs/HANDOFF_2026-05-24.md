# Handoff — romeos-display-v10 (24 Μαΐου 2026)

## Κατάσταση

Το firmware **λειτουργεί** στην οθόνη Viewe (COM3, env `BOARD_VIEWE_V10`). Ο χρήστης επιβεβαίωσε ότι όλα δουλεύουν σωστά μετά το τελευταίο upload.

**Build banner:** `BUILD 2026-05-24-v10-mic-touch`

## Τι υλοποιήθηκε σε αυτή τη φάση

### UI / οθόνες

- EEZ export 800×480: PRYMARY, MENU, HEAT, BOILER, LIGHT, BLINDS
- MENU hub navigation (κουμπιά EEZ → `loadScreen`)
- BACK: `button_menu_*` + εικονίδια `back_*`
- `ui.c`: `lv_scr_load` + `romeos_eez_runtime_on_screen_loaded()` για συγχρονισμό χρωμάτων

### Διορθώσεις που έγιναν

| Θέμα | Λύση |
|------|------|
| Ρολόι/ημερομηνία χαλάνε το MENU | Ξεχωριστά `obj` ανά οθόνη — PRYMARY `18/20/28–30`, MENU `41/42/54–56` |
| Εικονίδιο ΛΕΒΗΤΑ (hub) ≠ EEZ | Hub + οθόνη boiler → `img_boiler1` · status «Λεβήτα» → `img_boiler` |
| Εξωτερική = εσωτερική θερμοκρασία | Εσωτερική μόνο από SHT οθόνης · εξωτερική από `outdoor_c_x10` (UDP) |
| ALARM χρώμα | `bg_img_recolor` σε `lv_img` (όχι `img_recolor`) |
| «Σπίτι» χρώμα | `obj26` (PRYMARY), `obj52` (MENU) → `#008cec` όταν home Wi‑Fi |
| MIC δεν απαντούσε | Toggle + κόκκινο icon/label · bind σε κουμπί + εικονίδιο + κείμενο · fix hit-test (opa) |

### Υποδομή (από v5)

- Οικιακό Wi‑Fi NVS + `secrets/wifi_home.ini`
- NTP (Ελλάδα)
- `romeos_display_link` UDP προς μητρική
- Setpoint bar `obj7` (10–30 °C)
- Buzzer alarm GPIO 17

## MIC — τρέχουσα συμπεριφορά

- **Πάτημα:** toggle ON/OFF
- **ON:** εικονίδιο + κείμενο «MIC» κόκκινα
- **OFF:** επαναφορά μπλε/label χρώμα
- **Φωνή:** stub — μόνο `Serial.printf("[eez] MIC ON/OFF")` · δεν υπάρχει ακόμα I2S/ASR

Κώδικας: `mic_bind_all_screens()`, `mic_set_visual()` στο `romeos_eez_runtime.cpp`.

## Επόμενα βήματα (όταν συνεχίσετε)
