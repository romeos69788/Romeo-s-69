## 2. Τι ισχύει τώρα (σύντομη μνήμη)

### Wi‑Fi οικίας + `RomeosMB`

- **Οθόνη / μητρική:** STA στο οικιακό SSID από NVS (`romeos_wifi_nvs`)· εντολές Serial `WIFI_HOME_SET` / `WIFI_HOME_SHOW` / `WIFI_HOME_CLR` / `WIFI_HOME_HELP` — λεπτομέρειες **§9.3** στο [`HANDOFF_2026-04-13.md`](./HANDOFF_2026-04-13.md).
- **Κωδικός δεν μπαίνει σε `platformio.ini`**· μόνο NVS μέσω Serial ή script `scripts/set-wifi-home.ps1` (κλείσε πρώτα Serial Monitor στο IDE).
- **ESP32 / S3 = συνήθως 2.4 GHz.** Αν το router δίνει **μόνο 5 GHz** στο ίδιο όνομα SSID, το STA μπορεί να αποτυγχάνει — έλεγχος από admin router.
- **LAN:** στην οθόνη, `ROMEOS_MB_LAN_IP` στο `platformio.ini` = **IP μητρικής** στο ίδιο subnet (όχι IP PC).

### Ενδείξεις `ui_Romeos1` (HOME vs ESP32)

- **HOME πράσινο + παλμός τόξων:** συνδεδεμένο STA στο **αποθηκευμένο** home SSID (`romeos_display_link_home_wifi_assoc()`).
- **ESP32 πράσινο + παλμός:** **Wi‑Fi συνδεδεμένο** και **πρόσφατα UDP** από μητρική (`romeos_display_link_is_connected()`).
- **Συχνό σενάριο:** HOME πράσινο, ESP32 κόκκινο μετά από λίγα δευτερόλεπτα → **χάθηκε το UDP** (LAN IP, μητρική off-LAN, firewall)· **όχι** απαραίτητα λάθος κωδικός Wi‑Fi.
- Λεπτομέρειες: [`HANDOFF_2026-04-13.md`](./HANDOFF_2026-04-13.md) **§9.6**, κώδικας `thermostat-ui-demo/src/main.cpp` (`squareline_wifi_link_status_timer_cb`).

### Ρολόι & NTP (internet)

- **SNTP** (`romeos_ntp.cpp`): `pool.ntp.org`, `time.google.com`, ζώνη **EET-2 / EEST-3** (ρητό `EET-2EEST-3,...`).
- **Συγχρόνιση με internet** μόνο όταν υπάρχει **διαδρομή** προς internet (συχνά **όχι** σε καθαρό `RomeosMB` χωρίς default route)· μετά από ~60 s χωρίς sync, Serial: προειδοποίηση `[romeos_ntp] no NTP sync after 60s...`.
- **Χωρίς NTP:** ρολόι από NVS + monotonic (`main.cpp` — `squareline_clock_*`).

### PlatformIO (Windows)

- **`framework-arduinoespressif32-libs`:** χωρίς `package.json` στο zip Espressif → `MissingPackageManifestError`· λύση: ελάχιστο `package.json` στο global `packages/framework-arduinoespressif32-libs/` ή επανεγκατάσταση + ξανά manifest — **§9.4**.
- **`thermostat-ui-demo`:** αφαιρέθηκε override `esp32-3.1.1-h.zip` (ελλιπές για S3)· default πλήρες libs από platform.
- **`build_src_filter`:** `+<../../shared/src/romeos_wifi_nvs.cpp>` (βάση **`src/`**).

## 4. Προτεινόμενα επόμενα (όταν συνεχίσεις)

1. Επιβεβαίωση **IP μητρικής** στο LAN και **`ROMEOS_MB_LAN_IP`** στην οθόνη.
2. Αν χρειάζεται **RGB drift** χωρίς `h.zip`, εξερεύνηση ρυθμίσεων `board_override` / timing (χωρίς ελλιπές libs zip).
3. Συνέχεια πίνακα `motherboard-fw/README.md` (αισθητήρες, `ROMEOS_MB_TELEMETRY_OK`, κ.λπ.).
