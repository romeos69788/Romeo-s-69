Σημειώσεις από τη συζήτηση ώστε να συνεχιστεί η δουλειά χωρίς απώλεια context. Δεν αντικαθιστά τα λεπτομερή έγγραφα — παραπέμπει σε αυτά.

## 1. Κλέμες / αισθητήρες (τεκμηρίωση)

- **Πλήρης πίνακας + στρατηγική U5 (fake «εσωτερικός αέρας» προς inverter):**  
  **`CONNECTORS_AND_INDOOR_AIR_SPOOF.md`**
- Εκεί περιλαμβάνονται: LCD 2004 (I2C), **CN5** DHT εξωτερικού, **CN4** πέντε DS18B20, **U6** ροή, **U5** προσομοίωση για οικονομία inverter (~10 / ~20 / >~30 °C συνθετική «θερμοκρασία χώρου» ανάλογα με νερό/setpoint — ρυθμιζόμενα στο firmware).
- **U4** (*backup therm.control*): αριστερά πάνω γωνία, κάτω από τρύπα στήριξης· εφεδρική/μελλοντική είσοδος· firmware `PIN_BACKUP_THERM = GPIO39`· **αφήνεται ως έχει** μέχρι να οριστεί χρήση.
- **`REVIEW_board_v1_2026-05-07.md`:** ενημερωμένο με παραπομπή στο παραπάνω αρχείο και αναφορά **U4**.

## 2. Αισθητήρας φωτός vs ιεράρχηση πηγών

- Συμφωνήθηκε: **δεν βασίζουμε** την απόφαση «μέρα = ηλιακός» κυρίως σε φως — η **θερμοκρασία κυκλώματος** (ηλιακός, μπόιλερ, δάπεδο) είναι η αξιόπιστη βάση για **ηλιακός → αντλία θερμότητας → αντίσταση 4 kW (300 L)**.
- **Αισθητήρας φωτός:** προαιρετικό μόνο για UI/καταγραφή — **όχι** ως εφεδρεία αν χαλάσει θερμοκρασίας νερού.
- **Εφεδρεία αισθητήρων:** με **πολλαπλούς DS18B20** στο υδραυλικό — ορισμός **κύριων / εφεδρικών** ρόλων στο firmware· υστέρηση και επιβεβαίωση πριν αλλαγή σταδίων· σημείωση ότι βλάβη στο **ίδιο bus 1-Wire** μπορεί να πάρει όλους μαζί (η εφεδρεία καλύπτει κυρίως χαλασμένο probe).

---

## 3. Οθόνη μητρικής: από LCD 2004 σε HMI 5″

- Το **LCD 2004** περιορίζεται σε 4 γραμμές· για πλούσιο ταμπλό στο λεβητοστάσιο συμφωνήθηκε **ξεχωριστό module ~5″ IPS** με **ESP32‑S3**, αφή, PSRAM, LVGL — οικονομική επιλογή της τάξης **~27 €** (όπως στο προϊόν που κοινοποιήθηκε).
- Το module είναι **δεύτερος ESP** — **όχι** αντικατάσταση του ESP της μητρικής για ρελέ/αισθητήρες· ρόλος **HMI μόνο**.

---

## 4. Σύνδεση μητρικής ↔ οθόνη 5″ ↔ θερμοστάτης 7″

- **Θερμοστάτης 7″:** παραμένει **άλλη συσκευή** (ίδια οικογένεια UI)· **ασύρματο** (Wi‑Fi / ίδιο πρωτόκολλο με σήμερα — π.χ. UDP/MQTT ανά υλοποίηση).
- **Οθόνη 5″ στο καπάκι κουτιού μητρικής:** **ενσύρματη προτίμηση — UART** προς την μητρική (κοινό GND, σύντομο καλώδιο, προσοχή EMI κοντά σε ρελέ). Ο λόγος είναι κυρίως **αξιοπιστία τοπικού ταμπλό** χωρίς εξάρτηση από router — **όχι** μεγάλη εξοικονόμηση CPU στη μητρική.
- Η μητρική συνεχίζει να εκτελεί τον έλεγχο· μπορεί να **εξάγει την ίδια λογική κατάσταση** και σε MQTT (για 7″) και σε UART (για 5″) — ένας κανόνας δεδομένων, δύο κανάλια εξόδου.

## 5. Αλλαγές PCB / firmware (όταν μπουν στην ουρά υλοποίησης)

**Ενημέρωση προς παραγγελία PCB:** η **κλέμα LCD 2004** και η **U4 (backup)** **παραμένουν στο PCB** όπως είναι — δεν αφαιρούνται για αυτό το gerber.

| Περιοχή | Δράση |
|--------|--------|
| **PCB** | Διατήρηση **LCD 2004 (I2C)** και **U4**· διατήρηση **I2C** για **RTC U9**· **προσθήκη** κλέμας **MIC_I2S** για μικρόφωνο **INMP441** (δες **`MIC_INMP441_I2S.md`**)· κλείδωμα **3× GPIO** για I2S στο EasyEDA (το `pin_map.h` είναι γεμάτο — απαιτείται επιλογή pins στο schematic). |
| **Παραγγελία BOM** | **2×** ίδιο module **INMP441 I2S** — ένα μητρική, ένα θερμοστάτης 7″. |
| **Επόμενο βήμα** | Προαιρετικά **UART header** (TX/RX + GND) προς οθόνη 5″ στο καπάκι — επιλογή GPIO χωρίς σύγκρουση με ρελέ/SPI/I2C/strapping. |
| **`motherboard-fw`** | Όταν χρειάζεται: flag για LCD αν δεν populated· ενημέρωση `pin_map.h` για I2S μικροφώνου· τηλεμετρία/HMI όπως είχε συζητηθεί. |
| **Firmware 5″** | Ξεχωριστό project — πελάτης UART (και προαιρετικά Wi‑Fi). |
