# Δύο οθόνες Viewe — ρόλοι (συζήτηση 2026-07-19)

> **Κατάσταση:** υπόθεση εργασίας · **όχι** πλήρες κλείδωμα UI.  
> Λεπτομέρειες panel (τι φαίνεται / τι ελέγχεται) συζητιούνται · κλείδωμα μόνο με ρητή εντολή χρήστη.

**Κατεύθυνση χρήστη:** Υπάρχουν **δύο** Viewe 7″ (ίδιο hardware).  
Η **παλιά** → οθόνη δίπλα στη **μητρική Alpha** (λεβητοστάσιο / plant).  
Η **νέα** → **θερμοστάτης χώρου** (setpoint · φώτα · ρολά · comfort).

## Οθόνη 1 — Panel Alpha (αυτή που διαμορφώνουμε τώρα)

### Hardware link (κλειδωμένο)

| CN_PANEL pin | Net | Alpha |
|--------------|-----|-------|
| 1 | GND | GND |
| 2 | 5V | 5V_AUX / 5V_ESP |
| 3 | TX | **GPIO25** (PANEL_TX) |
| 4 | RX | **GPIO33** (PANEL_RX) |

- **115200 8N1** · crossover: Alpha TX → Viewe RX · Alpha RX ← Viewe TX  
- **Όχι** σύνδεση room οθόνης στο CN_PANEL

### Τι δείχνει / κάνει (υπό συζήτηση — όχι lock)

| Περιοχή | Κατεύθυνση χρήστη | Σημείωση agent |
|---------|-------------------|----------------|
| **Plant temps** | νερό εισαγωγή/προσαγωγή · εναλλάκτης in/out · HP | Ναι για panel · λίστα probes TBD |
| **HP / λειτουργία** | κατάσταση αντλίας κ.λπ. | + defrost / fault αν υπάρχει σήμα |
| **Ρελέ Alpha/Beta** | βλέπω + ελέγχω μητρικές | Τι ακριβώς = «έλεγχος» → ανοιχτό |
| **Φώτα / ρολά** | **όχι** στο panel → room | Συμφωνία |
| **Setpoint χώρου** | όχι κύριο χειριστήριο εδώ | Ένδειξη μόνο; ανοιχτό |

### Βάση firmware

- Ξεκίνημα από **romeos-display-v10** (EEZ: PRYMARY / MENU / HEAT / BOILER / …)
- Νέο role flag: `ROMEOS_DISPLAY_ROLE_ALPHA_PANEL`
- Πρωτεύον link: **UART** στο CN_PANEL (όχι μόνο UDP SoftAP)
- UI: έμφαση οθόνη **BOILER** / νέα **SYSTEM** · HEAT/LIGHT/BLINDS δευτερεύοντα ή κρυφά στο panel role

## UX panel — υπό συζήτηση (2026-07-19, μήνυμα 9)

> **Όχι lock.** Κατεύθυνση χρήστη για συζήτηση.

### Αρχή

- **Σβήσιμο** παλιού UI θερμοστάτη χώρου στην οθόνη λεβητοστασίου
- **Χτίσιμο από την αρχή** όπως τη φαντάζεται ο χρήστης
- **Κύρια οθόνη:** 4–5 πλαίσια (γενική εικόνα)
- **Πάτημα πλαισίου → δεύτερη οθόνη** με αναλυτικές ενδείξεις / έλεγχο εκείνου του τομέα

### Παράδειγμα (χρήστης)

| Κύρια (πλαίσιο) | Δεύτερη οθόνη |
|-----------------|---------------|
| Θερμοκρασίες νερού | Όλα τα water probes: ενδοδαπέδια προσαγωγή/επιστροφή · εναλλάκτης in/out · μπόιλερ άνω/κάτω · διαχωριστής · ό,τι έχει αισθητήρα |
| Λειτουργία αντλίας | Κατάσταση HP · κατανάλωση ρεύματος · «τα πάντα» του τομέα |

### Ανοιχτά

- Ποια ακριβώς 4–5 πλαίσια στην κύρια
- Ποιοι αισθητήρες υπάρχουν **ήδη** στο hardware vs μελλοντικοί
- Πηγή kWh/ρεύματος HP (TUYA / CT Beta)
- Αν οι έξοδοι/ρελέ είναι ξεχωριστό πλαίσιο

## Αποφάσεις ανοιχτές (να κλείσουμε στη φάση B)

1. Η panel οθόνη κρατάει **χειροκίνητο setpoint νερού / mode** ή μόνο monitoring;
